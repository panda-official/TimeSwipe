Measured values transformation control

This API allows to control how the values, measured in mV must be transformed.

Parameters translationOffsets and translationSlopes can be used to provide values for transformations of the values, measured in mV, by applying the following formula for that purpose:

data[i] = (value[i] - translationOffsets[i]) / translationSlopes[i]

where: i - is a sensor number, value - is a value, measured in mV by the sensor i.

panda::timeswipe::Driver_settings::Driver_settings ( std::string_view  json_text)
explicit

The constructor.

Parses the JSON input. Possible JSON members are:

  • sampleRate - an integer (see sample_rate());
  • burstBufferSize - an integer (see burst_buffer_size());
  • frequency - an integer (see frequency());
  • translationOffsets - an array of integers (see translation_offsets());
  • translationSlopes - an array of floats (see translation_slopes()). The exception with code Errc::driver_settings_invalid will be thrown if both burstBufferSize and frequency are presents in the same JSON input.

Driver_settings & panda::timeswipe::Driver_settings::set_burst_buffer_size ( std::optional< std::size_t >  size)

Sets the burst buffer size.

Requires
!size || (Driver::instance().min_sample_rate() <= *size && *size <= Driver::instance().max_sample_rate()).
Effects
Affects the values returned by frequency() and to_json_text(). (The later will be without the frequency member.)
Parameters
sizeThe number of records that the driver should deliver to Driver::Data_handler. std::nullopt means "as much as provided by the board".
Warning
This setting can be applied with Driver::set_driver_settings() only if !Driver::instance().is_measurement_started(true).

◆ set_sample_rate()

Driver_settings & panda::timeswipe::Driver_settings::set_sample_rate ( std::optional< int >  rate)

Set sample rate.

If this setting isn't set, the driver will use Driver::instance().max_sample_rate().

Parameters
rateThe value of sample rate.
Returns
The reference to this instance.
Requires
!rate || (Driver::instace().min_sample_rate() <= *rate && *rate <= Driver::instance().max_sample_rate()).
Warning
It's highly recommended to use rate for which (Driver::instance().max_sample_rate() % rate == 0) for best performance! In other words, the lower the value of std::gcd(Driver::instance().max_sample_rate(), rate), the worse the performance of the resampling.
This setting can be applied with Driver::set_driver_settings() only if !Driver::instance().is_measurement_started(true).
See also
sample_rate(), Driver::min_sample_rate(), Driver::max_sample_rate().
